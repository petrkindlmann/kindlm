# Plan 02-02: GitHub Action + HTTP Provider Adapter

## Goal

Create a GitHub Action for PR test comments and a generic HTTP provider adapter for custom API endpoints.

## Requirements

- **CLI-V2-04**: GitHub Action that posts test results as PR comment
- **CLI-V2-05**: Generic HTTP provider adapter for custom APIs

## Wave 1 — no dependencies

---

## Tasks

### Task 1 — Create HTTP provider adapter in core

**New file:** `packages/core/src/providers/http.ts`

Factory: `createHttpProviderAdapter(httpClient, providerConfig)`. Config: `{method, headers, bodyTemplate, responseMapping}`.

Template interpolation: `{{model}}`, `{{messages}}`, `{{temperature}}`, `{{maxTokens}}`, `{{topP}}`.
Response mapping: `getByPath(obj, "choices[0].message.content")` dot-path extraction.
Headers: `{{apiKey}}` placeholder for auth.

Uses injected HttpClient (zero I/O). Wraps with `withRetry()` from retry.ts. Returns ProviderResponse with text, usage, latencyMs.

**Export:** `getByPath` for testing. Export types from `providers/index.ts`.
**Test:** `packages/core/src/providers/http.test.ts`

### Task 2 — Add HTTP provider to config schema

**Modify:** `packages/core/src/config/schema.ts`

Add `HttpProviderConfigSchema` with: `apiKeyEnv?`, `baseUrl` (url), `method` (enum), `headers` (record), `bodyTemplate` (string), `responseMapping` ({text, inputTokens?, outputTokens?}).

Add `http: HttpProviderConfigSchema.optional()` to ProvidersSchema.
Add `"http"` to provider enum in ModelSchema.

### Task 3 — Register HTTP provider in registry

**Modify:** `packages/core/src/providers/registry.ts`

Special case `"http"` in `createProvider()` — requires `httpConfig` parameter. Update error message to include "http" in supported list.

### Task 4 — Wire HTTP provider into CLI

**Modify:** `packages/cli/src/utils/run-tests.ts`

Handle `name === "http"` in provider creation loop. Pass HTTP-specific config to `createProvider()`. Allow optional `apiKeyEnv` for HTTP provider.

### Task 5 — Create GitHub Action

**New file:** `/action.yml` (repo root)

Composite action:
1. Setup Node.js
2. Run `npx @kindlm/cli@latest test --reporter json`
3. Post PR comment via `actions/github-script` with pass/fail badge, metrics table
4. Upsert comment using `<!-- kindlm-results -->` marker
5. Respect `fail-on-error` input

Inputs: `config`, `node-version`, `comment`, `fail-on-error`

### Task 6 — Create example workflow

**New file:** `examples/github-action.yml`

Minimal example showing usage in a PR workflow.

### Task 7 — Update exports and verify

Update barrel exports in core. Run full test suite + typecheck + lint.
