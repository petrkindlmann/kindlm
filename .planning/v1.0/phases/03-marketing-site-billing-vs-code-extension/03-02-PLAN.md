## Objective

Replace inline `price_data` in the Stripe checkout route with pre-created Stripe Price IDs, add integration tests for the full checkout-to-webhook flow, and verify all three webhook event handlers work correctly with the Price ID approach.

## Context

- Billing routes exist in `packages/cloud/src/routes/billing.ts` with checkout, portal, and webhook endpoints.
- **Critical bug:** Lines 108-111 of `billing.ts` use inline `price_data` which creates a new anonymous Price on every checkout. This breaks Customer Portal (needs pre-created Prices) and prevents Stripe dashboard analytics.
- The `subscription.updated` webhook reads plan from `items.data[0].price.metadata.plan` — this is dead code with inline `price_data` (no metadata set). Once pre-created Prices have `plan` metadata, this path will work.
- Webhook signature verification is implemented correctly (HMAC-SHA256, timestamp replay protection, constant-time comparison).
- 11 existing tests in `billing.test.ts` cover basic routes and webhook events.
- `Bindings` type in `types.ts` currently has `STRIPE_SECRET_KEY?` and `STRIPE_WEBHOOK_SECRET?` — needs `STRIPE_TEAM_PRICE_ID?` and `STRIPE_ENTERPRISE_PRICE_ID?`.

## Manual Prerequisites (autonomous: false)

Before executing the code tasks below, the following manual steps must be completed in the Stripe dashboard:

### Step 1: Create Stripe Products and Prices (Stripe Dashboard)

1. Log in to [Stripe Dashboard](https://dashboard.stripe.com) (use test mode for initial verification).
2. Go to **Products** > **Add product**.
3. Create **KindLM Team**:
   - Name: `KindLM Team`
   - Price: $49.00 / month (recurring)
   - In the Price's metadata, add key `plan` with value `team`
   - Copy the Price ID (starts with `price_`)
4. Create **KindLM Enterprise**:
   - Name: `KindLM Enterprise`
   - Price: $299.00 / month (recurring)
   - In the Price's metadata, add key `plan` with value `enterprise`
   - Copy the Price ID (starts with `price_`)

### Step 2: Set Worker Secrets

```bash
# Set the Price IDs as Worker secrets (use test mode IDs first, then live)
npx wrangler secret put STRIPE_TEAM_PRICE_ID
# Paste: price_xxxxx (from step 1)

npx wrangler secret put STRIPE_ENTERPRISE_PRICE_ID
# Paste: price_xxxxx (from step 1)
```

### Step 3: Configure Customer Portal (Stripe Dashboard)

1. Go to **Settings** > **Billing** > **Customer portal**.
2. Under **Products**, add both KindLM Team and KindLM Enterprise products.
3. Enable "Allow customers to switch plans" and "Allow customers to cancel subscriptions".
4. Save the portal configuration.
