Add this block between the CORS middleware closing `});` and the `// Rate-limit on auth routes` comment:

```typescript

app.use("*", async (c, next) => {
  if (c.req.path === "/health" || c.req.path === "/") {
    return next();
  }
  const required = ["GITHUB_CLIENT_ID", "GITHUB_CLIENT_SECRET", "SIGNING_KEY_SECRET"] as const;

  if (missing.length > 0) {
    // eslint-disable-next-line no-console
    console.error(`Missing required secrets: ${missing.join(", ")}`);
    return c.json({ error: "Server misconfigured" }, 500);
  }
  return next();
});
```

```typescript
const TOKEN_EXPIRY_DAYS = 90;
```

Then replace the `createToken` call (lines 249-257). Change the `null` for `expiresAt` (the 6th positional argument) to a computed date:

Replace:
```typescript
  await queries.createToken(
    org.id,

  );
```

With:
```typescript

    `login-${new Date().toISOString().slice(0, 10)}`,
    tokenHash,
    "full",
    null,
    tokenExpiresAt,
    user.id,
  );
```

Verify the parameter order by reading `queries.ts` `createToken` function. The parameters should be:
`(orgId, name, tokenHash, scope, projectId, expiresAt, userId)`

The 5th param is `projectId` (null for login tokens), 6th is `expiresAt` (was null, now set to 90 days).
