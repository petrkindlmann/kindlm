<title>Deploy marketing site to Cloudflare Pages (SITE-01)</title>
<read_first>
- site/out/index.html (verify build output exists from task 2)
- site/out/_headers (verify cache headers file was copied)
</read_first>
<action>
Create the Cloudflare Pages project and deploy:

```bash
cd /Users/petr/projects/kindlm

# Create the Pages project (if it doesn't exist yet)
npx wrangler pages project create kindlm-site

# Deploy the static output
npx wrangler pages deploy site/out --project-name=kindlm-site
```

After deployment, Wrangler will output the deployment URL (e.g., `https://abc123.kindlm-site.pages.dev`).

**Configure custom domain `kindlm.com`:**

```bash
TOKEN="$(grep CLOUDFLARE_API_TOKEN ~/projects/ai-news-monitor/.env.local | cut -d= -f2)"
ZONE=$(curl -s "https://api.cloudflare.com/client/v4/zones?name=kindlm.com" \
  -H "Authorization: Bearer $TOKEN" | python3 -c "import json,sys; print(json.load(sys.stdin)['result'][0]['id'])")

# Create proxied A record for kindlm.com
curl -s -X POST "https://api.cloudflare.com/client/v4/zones/$ZONE/dns_records" \
  -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  -d '{"type":"A","name":"kindlm.com","content":"192.0.2.1","proxied":true}'

# Create www CNAME redirect
curl -s -X POST "https://api.cloudflare.com/client/v4/zones/$ZONE/dns_records" \
  -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  -d '{"type":"CNAME","name":"www.kindlm.com","content":"kindlm.com","proxied":true}'
```

Then add `kindlm.com` as a custom domain in the Cloudflare Pages dashboard under Pages > kindlm-site > Custom domains > Add custom domain.
</action>
