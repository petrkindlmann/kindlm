# Plan 04-01: Sentry Error Tracking + v2.0.0 Release

## Goal

Add Sentry error tracking to Cloud Workers and cut the v2.0.0 release with all v2 features.

## Requirements

- **INFRA-04**: Sentry error tracking for Workers

## Wave 1

---

## Tasks

### Task 1 — Add @sentry/cloudflare dependency

**Modify:** `packages/cloud/package.json`
Add `"@sentry/cloudflare": "^9.0.0"` to dependencies. `npm install`.

### Task 2 — Wrap Worker with Sentry

**Modify:** `packages/cloud/src/index.ts`

Import and wrap the Hono app export:

```typescript
import * as Sentry from "@sentry/cloudflare";
```

Wrap the default export:
```typescript
export default Sentry.withSentry(
  (env) => ({
    dsn: env.SENTRY_DSN,
    tracesSampleRate: 0.1,
  }),
  app,
);
```

Add `SENTRY_DSN` to the `Bindings` interface in `types.ts`:
```typescript
SENTRY_DSN?: string;
```

The `withSentry` wrapper catches unhandled errors, reports them to Sentry with request context, and re-throws.

### Task 3 — Set SENTRY_DSN Worker secret

**Manual checkpoint:**
1. Create Sentry project at sentry.io (Cloudflare Workers platform)
2. Copy DSN
3. `cd packages/cloud && npx wrangler secret put SENTRY_DSN`
