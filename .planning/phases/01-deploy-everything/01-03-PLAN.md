<task type="checkpoint:human-action" gate="non-blocking">
  <name>Task 1: Set CF_API_TOKEN in GitHub Actions secrets</name>
  <action>
**Why:** The Deploy Cloud GitHub Actions workflow failed because `secrets.CF_API_TOKEN` is not set. Future pushes to `packages/cloud/**` will trigger this workflow and fail without it.

**Steps:**

1. Go to: https://github.com/petrkindlmann/kindlm/settings/secrets/actions
2. Click "New repository secret"
3. Name: `CF_API_TOKEN`
4. Value: Your Cloudflare API token (the one Wrangler uses locally — check `~/Library/Preferences/.wrangler/config/default.toml` or create a new one at https://dash.cloudflare.com/profile/api-tokens with "Edit Cloudflare Workers" template)
5. Click "Add secret"

**Required permissions for the token:**
- Account: Cloudflare Workers Scripts (Edit)
- Account: Cloudflare Workers KV Storage (Edit)
- Account: Cloudflare Workers R2 Storage (Edit)
- Account: D1 (Edit)
- Zone: kindlm.com (Edit Workers Routes)

**Verify:** After setting, push a trivial change to `packages/cloud/` or re-run the failed workflow:
```bash
gh workflow run "Deploy Cloud" --ref main
```

<task type="checkpoint:human-action" gate="non-blocking">
  <name>Task 2: Set SENTRY_DSN Worker secret (OPS-06)</name>
  <action>
**Why:** The Cloud Worker has Sentry error monitoring code (`@sentry/cloudflare`) but needs the DSN to connect to a Sentry project.

**Steps:**

1. Go to https://sentry.io and log in (or create account)
2. Create a new project:
   - Platform: "Cloudflare Workers" (or "JavaScript")
   - Project name: `kindlm-api`
   - Team: (your team or default)
3. Copy the DSN from the project settings (looks like `https://xxx@xxx.ingest.sentry.io/xxx`)
4. Set it as a Wrangler secret:
   ```bash
   cd packages/cloud && npx wrangler secret put SENTRY_DSN
   ```
   Paste the DSN when prompted.
5. Verify it's set:
   ```bash
   npx wrangler secret list
   ```
   Should show `SENTRY_DSN` in the list.

**Optional but recommended:** Also set for staging:
```bash
cd packages/cloud && npx wrangler secret put SENTRY_DSN --env staging
```
  </action>
  <acceptance_criteria>
    - `npx wrangler secret list` (run from packages/cloud/) includes SENTRY_DSN
    - Sentry project "kindlm-api" exists at sentry.io
  </acceptance_criteria>
  <resume-signal>Type "sentry done" when the DSN is set</resume-signal>
  <done>SENTRY_DSN is set as a Worker secret. Sentry error monitoring is active for kindlm-api.</done>
</task>

<task type="checkpoint:human-action" gate="non-blocking">
  <name>Task 3: Publish VS Code extension (OPS-07)</name>
  <action>
**Why:** The VS Code extension has been updated (README, package.json, schema, snippets, completions, hover) and needs to be published to the VS Code Marketplace.

**Prerequisites:**
- A Personal Access Token (PAT) from Azure DevOps with "Manage" scope on Marketplace
- Publisher registered at https://marketplace.visualstudio.com/manage

**Steps:**

1. If you don't have a PAT yet:
   - Go to https://dev.azure.com/
   - Click "User Settings" (top-right) > "Personal access tokens"
   - Create new token with:
     - Organization: "All accessible organizations"
     - Scopes: "Marketplace > Manage"
   - Copy the token

2. Log in to vsce:
   ```bash
   npx vsce login kindlm
   ```
   Paste the PAT when prompted. (Replace `kindlm` with your publisher ID if different.)

3. Build and publish:
   ```bash
   cd packages/vscode
   npx vsce package
   npx vsce publish
   ```

4. Verify:
   - Check https://marketplace.visualstudio.com/items?itemName=kindlm.kindlm (replace with actual publisher.name)
   - Or run: `npx vsce show kindlm.kindlm` (replace with actual extension ID)

**Note:** Check `packages/vscode/package.json` for the publisher name and extension version before publishing. The `package.json` should have a `"publisher"` field.
  </action>
  <acceptance_criteria>
    - `npx vsce show` for the extension returns the latest version
    - Extension is visible on the VS Code Marketplace
  </acceptance_criteria>
  <resume-signal>Type "vscode done" when published</resume-signal>
  <done>VS Code extension published to Marketplace with latest schema, completions, and hover docs.</done>
</task>

<task type="checkpoint:human-action" gate="non-blocking">
  <name>Task 4: Create Stripe production products (OPS-08)</name>
  <action>
**Why:** KindLM Cloud has three pricing tiers that need corresponding Stripe products and prices for billing to work.

**Steps:**

1. Go to https://dashboard.stripe.com/products (make sure you're in LIVE mode, not test mode)

2. Create Product 1 — **KindLM Team**:
   - Name: `KindLM Team`
   - Description: `Team plan — up to 10 members, 5 projects, 90-day test history, compliance PDF export, Slack webhooks`
   - Pricing: Recurring, $49.00/month
   - After creation, copy the Price ID (starts with `price_`)

3. Create Product 2 — **KindLM Enterprise**:
   - Name: `KindLM Enterprise`
   - Description: `Enterprise plan — unlimited members & projects, unlimited history, signed compliance reports, SSO/SAML, audit log API, 99.9% SLA`
   - Pricing: Recurring, $299.00/month
   - After creation, copy the Price ID (starts with `price_`)

4. Set the Stripe secrets as Worker secrets:
   ```bash
   cd packages/cloud
   npx wrangler secret put STRIPE_SECRET_KEY
   # Paste your Stripe live secret key (sk_live_...)

   npx wrangler secret put STRIPE_WEBHOOK_SECRET
   # You'll get this after creating the webhook endpoint (next step)
   ```

5. Create a Stripe webhook endpoint:
   - Go to https://dashboard.stripe.com/webhooks
   - Click "Add endpoint"
   - URL: `https://api.kindlm.com/v1/billing/webhook`
   - Events to listen for:
     - `checkout.session.completed`
     - `customer.subscription.updated`
     - `customer.subscription.deleted`
     - `invoice.payment_succeeded`
     - `invoice.payment_failed`
   - After creation, copy the webhook signing secret (starts with `whsec_`)
   - Set it: `npx wrangler secret put STRIPE_WEBHOOK_SECRET` (paste the whsec_ value)

6. Note the Price IDs — these will need to be configured in the Cloud Worker's billing route or as environment variables. Check `packages/cloud/src/routes/billing.ts` for how price IDs are referenced.

**Verify:**
```bash
npx wrangler secret list
```
Should show: STRIPE_SECRET_KEY, STRIPE_WEBHOOK_SECRET
  </action>
  <acceptance_criteria>
    - Two products exist in Stripe dashboard: "KindLM Team" ($49/mo) and "KindLM Enterprise" ($299/mo)
    - `npx wrangler secret list` includes STRIPE_SECRET_KEY and STRIPE_WEBHOOK_SECRET
    - Webhook endpoint configured at https://api.kindlm.com/v1/billing/webhook
  </acceptance_criteria>
  <resume-signal>Type "stripe done" when products and secrets are configured</resume-signal>
