# Plan 01-01: v1 Cleanup — VS Code Publish & Stripe Products

## Goal

Close out the two remaining v1 items that require manual interaction with external services.

## Requirements

- **CLEAN-01**: VS Code extension published to marketplace
- **CLEAN-02**: Stripe Products/Prices created in production, Worker secrets set

## Wave

Wave 1 — no dependencies

---

## Tasks

### Task 1 — Fix VS Code package script

**File:** `packages/vscode/package.json`

Remove the `--allow-package-secrets slack` flag (was needed for workaround during v1 build) and add a publish script:

Change:
```json
"package": "npx vsce package --no-dependencies --allow-package-secrets slack"
```

To:
```json
"package": "npx vsce package --no-dependencies",
"publish": "npx vsce publish --no-dependencies"
```

Verify VSIX builds:
```bash
cd packages/vscode && npm run build && npm run package
```

---

### Task 2 — MANUAL CHECKPOINT: Publish VS Code Extension

1. Go to https://marketplace.visualstudio.com/manage
2. Sign in, create publisher ID `kindlm` if not exists
3. Create PAT at https://dev.azure.com → Personal Access Tokens (scope: Marketplace > Manage)
4. Publish:
```bash
cd packages/vscode && npx vsce publish -p <PAT>
```
5. Verify: `code --install-extension kindlm.kindlm`

---

### Task 3 — MANUAL CHECKPOINT: Create Stripe Products

In Stripe Dashboard (https://dashboard.stripe.com/products):

1. Product "KindLM Team" — $49/month recurring → copy `price_id`
2. Product "KindLM Enterprise" — $299/month recurring → copy `price_id`
3. Set Worker secrets:
```bash
cd packages/cloud
npx wrangler secret put STRIPE_TEAM_PRICE_ID
npx wrangler secret put STRIPE_ENTERPRISE_PRICE_ID
```
4. Configure webhook at `https://api.kindlm.com/billing/webhook` for events: `checkout.session.completed`, `customer.subscription.updated`, `customer.subscription.deleted`
