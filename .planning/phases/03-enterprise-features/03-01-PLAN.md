# Plan 03-01: SAML XML Parser + Signed Compliance Reports

## Goal
Replace regex-based SAML XML parsing with `fast-xml-parser` for security, add ECDSA support, and enhance compliance signing with run-level integration.

## Requirements
- **ENT-01**: SAML signature verification with proper XML parser
- **ENT-02**: Signed compliance PDF reports

## Wave 1

---

## Tasks

### Task 1 — Add fast-xml-parser dependency
Add `"fast-xml-parser": "^5.0.0"` to packages/cloud/package.json. `npm install`.

### Task 2 — Create SAML XML parser wrapper
**New:** `packages/cloud/src/saml/xml-parser.ts`
- `parseSamlResponse(xml)` — parse XML to typed object
- `findElementById(doc, id)` — XSW-safe element lookup (verify only one match)
- `extractTextContent(doc, tagPath)` — dot-path text extraction
- `extractAttributeValue(doc, attrName)` — SAML attribute extraction
- `serializeSignedInfo(doc)` — namespace-aware SignedInfo serialization
**Test:** `packages/cloud/src/saml/xml-parser.test.ts`

### Task 3 — Rewrite SAML extraction functions
**Modify:** `packages/cloud/src/saml/helpers.ts`
Replace regex in: extractNameID, extractAttribute, extractIssuer, extractAssertionId, checkConditions with xml-parser calls. Keep function signatures identical.

### Task 4 — Rewrite signature verification
**Modify:** `packages/cloud/src/saml/helpers.ts`
Rewrite verifySamlSignature: parse XML, extract SignedInfo from tree, findElementById for reference (XSW defense), compute digest, verify signature. Add ECDSA P-256 support. Remove canonicalizeSignedInfo and extractReferencedElement.

### Task 5 — Update SSO tests
**Modify:** `packages/cloud/src/routes/sso.test.ts`
Add: XSW attack test, ECDSA test, namespace variation test, XML comment injection test.

### Task 6 — D1 migration for compliance signing
**New:** `packages/cloud/migrations/0012_compliance_signing.sql`
```sql
ALTER TABLE runs ADD COLUMN compliance_signature TEXT;
ALTER TABLE runs ADD COLUMN compliance_signed_at TEXT;
```

### Task 7 — Add run-level compliance signing endpoint
**Modify:** `packages/cloud/src/routes/compliance.ts`
- `POST /sign-report` — fetch run, sign with Ed25519 + HMAC-SHA256, store signature on run
- `POST /verify-report` — fetch run, verify both Ed25519 and HMAC

### Task 8 — Update compliance tests
Add: sign-report, verify-report, tampered report detection, round-trip test.

## Verification
```bash
npx turbo run test && npx turbo run typecheck
```
All existing tests pass, XSW attack detected, ECDSA works, compliance signing round-trips.
