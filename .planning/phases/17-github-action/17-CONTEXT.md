# Phase 17: GitHub Action - Context

**Gathered:** 2026-04-03
**Status:** Ready for planning

<domain>
## Phase Boundary

Create a published GitHub Action (`kindlm/test@v2`) that runs KindLM tests in CI workflows. The action installs `@kindlm/cli`, runs tests, uploads JUnit artifacts, optionally posts PR comments and uploads to KindLM Cloud. This is a NEW separate repo — no existing code to extend.

</domain>

<decisions>
## Implementation Decisions

### Repo structure (ACTION-07)
- **D-01:** Separate repo `kindlm/test-action` (not in the monorepo). GitHub requires `action.yml` at repo root and `dist/` checked in for JS actions.
- **D-02:** `runs.using: node20` — JS action, NOT Docker. Works on ubuntu, macos, windows.
- **D-03:** Bundle with `@vercel/ncc` into single `dist/index.js`. All deps bundled.
- **D-04:** Structure:
  ```
  kindlm/test-action/
  ├── action.yml
  ├── src/index.ts
  ├── dist/index.js      # bundled, checked in
  ├── package.json
  ├── tsconfig.json
  └── .github/workflows/test.yml  # action's own CI
  ```

### Inputs (ACTION-02)
- **D-05:** `config` — path to kindlm.yaml (default: `kindlm.yaml`)
- **D-06:** `reporter` — reporter type (default: `junit`)
- **D-07:** `args` — additional CLI args passed through to `kindlm test`
- **D-08:** `cloud-token` — optional `KINDLM_API_TOKEN` for cloud upload
- **D-09:** `comment` — post PR comment with summary (default: `true`)
- **D-10:** `version` — @kindlm/cli version to install (default: `latest`)

### Outputs (ACTION-03)
- **D-11:** `pass-rate` — percentage as string (e.g., "95.5")
- **D-12:** `total`, `passed`, `failed` — integer counts as strings
- **D-13:** `exit-code` — "0" or "1"

### Implementation flow
- **D-14:** Action flow: (1) Install `@kindlm/cli@{version}` globally via npm, (2) Run `kindlm test --reporter junit --reporter json {args}`, (3) Parse JSON output for counts, (4) Upload JUnit XML as artifact, (5) Post PR comment if enabled, (6) Upload to cloud if token set, (7) Set outputs.
- **D-15:** Use `@actions/core` for inputs/outputs/logging, `@actions/exec` for running CLI, `@actions/artifact` for JUnit upload, `@actions/github` for PR comments.

### PR comment format (ACTION-05)
- **D-16:** Markdown comment body:
  ```markdown
  ## KindLM Test Results

  | Status | Count |
  |--------|-------|
  | ✅ Passed | {N} |
  | ❌ Failed | {N} |
  | Total | {N} |

  **Pass rate:** {X}%

  {If failures:}
  ### Failing Tests
  - `{test-name}`: {failure reason}

  <sub>Generated by [KindLM](https://kindlm.com) · [Report]({cloud-url})</sub>
  ```
- **D-17:** Update existing comment (don't create new on each push) — find by marker `<!-- kindlm-test-results -->`.

### Security (ACTION-08)
- **D-18:** NEVER write raw model responses to step summary or PR comments. Only structured data (pass/fail counts, test names, assertion results).
- **D-19:** JUnit XML artifact contains test names and assertion messages but NOT model response text.

### Cross-platform (ACTION-01)
- **D-20:** Action's own CI tests on ubuntu-latest, macos-latest, windows-latest matrix.
- **D-21:** No platform-specific code — pure Node.js, no shell scripts.

### Cloud upload (ACTION-06)
- **D-22:** When `cloud-token` is set, run `kindlm upload` after tests complete. Non-fatal on failure (cloud upload should not fail CI).

### Claude's Discretion
- Exact npm install command (global vs npx)
- Whether to use `@actions/tool-cache` for caching the CLI install
- Error handling strategy for network failures during install
- Whether to support `working-directory` input for monorepo users

</decisions>

## Existing Code Insights

### Reusable Assets
- No existing action code — this is greenfield in a new repo.
- `packages/core/src/reporters/json.ts` — JSON output format that the action will parse for counts.
- `.github/workflows/ci.yml` — Reference for how the monorepo CI is structured (cross-platform matrix).

### Established Patterns
- KindLM CLI is installed via npm: `npm install -g @kindlm/cli` or `npx @kindlm/cli`
- JSON reporter outputs structured results to stdout
- JUnit reporter writes XML for CI integration
- `kindlm upload` pushes results to cloud

### Integration Points
- Action installs and wraps the published `@kindlm/cli` npm package
- No direct code dependency on the monorepo — action is a consumer of the published CLI
