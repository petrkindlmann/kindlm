# Plan 02-01: CLI Flags + Caching

## Goal

Add `--dry-run` (print test plan without API calls), `--watch` (re-run on config change), and result caching (`--no-cache` to bypass) to the CLI.

## Requirements

- **CLI-V2-01**: `--dry-run` mode
- **CLI-V2-02**: `--watch` mode
- **CLI-V2-03**: Result caching

## Wave 1 — no dependencies

---

## Tasks

### Task 1 — Create `buildTestPlan()` in core

**New file:** `packages/core/src/engine/test-plan.ts`

Extract execution plan logic from runner.ts. Returns `TestPlan` with entries (test name, model, repeat, assertion types, skip status) + summary (total units, API calls, models list). Pure function, zero I/O.

**Export from:** `packages/core/src/engine/index.ts`
**Test:** `packages/core/src/engine/test-plan.test.ts`

### Task 2 — Create dry-run formatter in CLI

**New file:** `packages/cli/src/utils/dry-run.ts`

`formatTestPlan(plan: TestPlan): string` — renders table with chalk (test name, model, runs, assertions). Footer: "No API calls will be made."

**Test:** `packages/cli/src/utils/dry-run.test.ts`

### Task 3 — Add `--dry-run` flag to test command

**Modify:** `packages/cli/src/commands/test.ts`

Add `.option("--dry-run", "...")`. In handler: parse config, build test plan, print, exit 0. Early return before `runTests()`.

### Task 4 — Create file watcher utility

**New file:** `packages/cli/src/utils/watcher.ts`

`watchFile({filePath, debounceMs, onChange, onError}): {close()}` — uses `node:fs.watch`, manual debounce, ENOENT handling.

**Test:** `packages/cli/src/utils/watcher.test.ts` — mock fs.watch, test debounce

### Task 5 — Add `--watch` flag to test command

**Modify:** `packages/cli/src/commands/test.ts`

Extract core test execution into `executeTests()` helper. Watch loop: run once, then watch config file, re-run on change. Catch errors to keep watcher alive. SIGINT closes watcher.

### Task 6 — Create result cache utility

**New file:** `packages/cli/src/utils/cache.ts`

Cache key: SHA-256 of `{model, messages, params, tools}`. Storage: `.kindlm/cache/{hash}.json`. 24h TTL, 100 file cap, LRU eviction by mtime. Follow `last-run.ts` patterns (mode 0o700/0o600).

**Test:** `packages/cli/src/utils/cache.test.ts`

### Task 7 — Create caching adapter wrapper

**New file:** `packages/cli/src/utils/caching-adapter.ts`

`createCachingAdapter(inner: ProviderAdapter): ProviderAdapter` — checks cache before `inner.complete()`, writes on miss. Spreads all other methods from inner.

**Test:** `packages/cli/src/utils/caching-adapter.test.ts`

### Task 8 — Integrate cache + `--no-cache` into test command

**Modify:** `packages/cli/src/commands/test.ts` — add `--no-cache` option
**Modify:** `packages/cli/src/utils/run-tests.ts` — wrap adapters with `createCachingAdapter()` unless `noCache`

## Verification

```bash
npx turbo run test && npx turbo run typecheck && npx turbo run lint
```

- `kindlm test --dry-run` prints plan, exits 0, no API calls
- `kindlm test --watch` re-runs on config change
- Cache populates `.kindlm/cache/`, second run uses cached responses
- `--no-cache` bypasses cache
