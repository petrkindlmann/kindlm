       ```
       const wt = await createWorktree(slug);
       worktreeCleanup = wt.cleanup;
       originalCwd = process.cwd();

       // Copy config file + referenced schema files into worktree (per ISOLATE-01)
       const absConfigPath = resolve(originalCwd, options.config);
       const yamlContent = readFileSync(absConfigPath, "utf-8");
       const referencedPaths = extractConfigFilePaths(yamlContent)
         .map((p) => resolve(originalCwd, p));
       referencedPaths.push(absConfigPath); // always copy config itself
       try {
         await copyFilesToWorktree(wt.path, originalCwd, referencedPaths);
       } catch (copyErr) {
         // Config copy failure: warn + continue (per CONTEXT.md decision)
         const copyMsg = copyErr instanceof Error ? copyErr.message : String(copyErr);
         console.warn(chalk.yellow(`Warning: file copy failed (${copyMsg}). Some files may be missing in worktree.`));
       }

       process.chdir(wt.path);
       console.log(chalk.dim(`Worktree: ${wt.path}`));
       ```

    **IMPORTANT:** The `readFileSync` import already exists at line 2 of test.ts. The `resolve` import already exists at line 2 (`import { resolve, dirname } from "node:path"`). No new package imports needed.

    **IMPORTANT:** The `copyFilesToWorktree` call wraps errors in a try/catch with warn+continue, EXCEPT for WorktreeError (path escape) which should bubble up and be caught by the outer catch that degrades gracefully. Adjust: do NOT catch WorktreeError in the inner try/catch — only catch non-WorktreeError:
    ```
    try {

    } catch (copyErr) {
      if (copyErr instanceof WorktreeError) throw copyErr; // path escape — let outer catch handle
    ```
