## Objective

Verify the full `kindlm upload` pipeline end-to-end: CLI sends test results to the Cloud API, the API stores them in D1, and the dashboard displays them with pass/fail status and assertion details. Fix any integration bugs found during verification.

## Context

- The Cloud API is live at `api.kindlm.com` with all routes deployed (Phase 1)
- The upload logic in `packages/cli/src/cloud/upload.ts` creates project, suite, run, then batch-inserts results
- The dashboard pages at `/projects/[projectId]/runs/[runId]` use SWR to fetch from `/v1/runs/:runId` and `/v1/results/:runId/results`
- Dashboard is deployed (Plan 02-01 prerequisite)
- The CLI upload command reads from `.kindlm/last-run.json` (saved by `kindlm test`)

Run the full E2E pipeline: execute a test suite, then upload results to the live Cloud API.

**Step 1: Create a minimal test config (if no kindlm.yaml exists):**
```bash
cd /tmp/kindlm-e2e-test
mkdir -p /tmp/kindlm-e2e-test && cd /tmp/kindlm-e2e-test

npx @kindlm/cli init
```

Or create a minimal `kindlm.yaml` manually:
```yaml
version: "1"
defaults:
  provider: openai:gpt-4o-mini
  temperature: 0
  runs: 1

suites:
  - name: "upload-e2e-test"
    system_prompt: "You are a helpful assistant."
    tests:
      - name: "basic-response"
        input: "Say exactly: hello world"
        assert:
          - type: keywords_present
            value: ["hello", "world"]
```

**Step 2: Run the test:**
```bash
npx @kindlm/cli test
# Should exit 0 with pass output
# Should create .kindlm/last-run.json
```

Verify the last-run file was saved:
```bash
ls -la .kindlm/last-run.json
# Should exist and be non-empty
```

**Step 3: Upload results:**
```bash
# Ensure you have a token (from kindlm login or env var)
npx @kindlm/cli upload -p "e2e-upload-test"
# Expected: "Uploaded successfully."
# Expected: Run ID, Project name, Suite name printed
```

If the upload fails:
- **401 Unauthorized:** Token is invalid or expired. Run `kindlm login` to get a fresh token.
- **404 Not Found:** Suite or project creation may have failed. Check the error message.
- **500 Internal Server Error:** Cloud API issue. Check `npx wrangler tail` for error logs.

**Step 4: Record the run URL:**
After a successful upload, note the Run ID and Project ID. These will be verified in the dashboard in task 3.

The dashboard URL will be: `https://app.kindlm.com/projects/<projectId>/runs/<runId>`

- Check if the CLI is slow to finalize the run (the PATCH call in upload.ts)
