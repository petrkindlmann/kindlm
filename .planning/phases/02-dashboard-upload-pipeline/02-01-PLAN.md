## Objective

Deploy the Next.js static export dashboard to Cloudflare Pages at `app.kindlm.com`, ensure SPA deep links resolve correctly (no 404 on paths like `/projects/abc`), and configure cache headers so `index.html` is never stale while hashed assets are cached permanently.

- `_redirects` file already exists in `public/` with `/* /index.html 200`
- No `_headers` file exists yet (cache headers missing -- DASH-03 gap)
- Deploy workflow at `.github/workflows/deploy-dashboard.yml` already deploys to `kindlm-dashboard` Pages project using `out/` directory
- API base URL defaults to `https://api.kindlm.com` (set in next.config.mjs)
- The Cloud API is live and healthy (verified in Phase 1)

## Tasks

<task id="1">
<title>Create _headers file for Cloudflare Pages cache control (DASH-03)</title>
<read_first>
- packages/dashboard/public/_redirects (verify existing SPA fallback rule)
- packages/dashboard/next.config.mjs (confirm static export and env config)
</read_first>
<action>
Create `packages/dashboard/public/_headers` with the following exact content:

```
/index.html
  Cache-Control: no-cache

/_next/static/*
  Cache-Control: public, max-age=31536000, immutable
```

This tells Cloudflare Pages:
- `index.html`: Always revalidate with the origin (users get the latest deploy immediately)
- `/_next/static/*`: Hashed JS/CSS bundles are safe to cache forever (Next.js includes content hashes in filenames)

Cloudflare Pages uses the `_headers` file natively -- no Worker configuration needed.
</action>
<acceptance_criteria>
- File `packages/dashboard/public/_headers` exists
- File contains `/index.html` followed by `Cache-Control: no-cache`
- File contains `/_next/static/*` followed by `Cache-Control: public, max-age=31536000, immutable`
</acceptance_criteria>
</task>

<task id="2">
<title>Verify and harden the _redirects SPA fallback (DASH-02)</title>
<read_first>
- packages/dashboard/public/_redirects (current content: `/*  /index.html  200`)
</read_first>
<action>
The existing `_redirects` file contains `/*  /index.html  200` which is the correct Cloudflare Pages SPA fallback rule. Verify it has no trailing whitespace or BOM characters that could cause parsing issues.

The file must contain exactly:
```
/*  /index.html  200
```

No changes needed if the file already matches. If there are extra lines or characters, replace the entire content with the single rule above.

**Important:** Cloudflare Pages processes `_redirects` before `_headers`. The `/* /index.html 200` rule is a "splat" rewrite (status 200), meaning all unmatched paths serve `index.html` content. This enables client-side routing for paths like `/projects/abc`, `/projects/abc/runs/xyz`, and `/login`.
</action>
<acceptance_criteria>
- File `packages/dashboard/public/_redirects` contains `/*  /index.html  200`
- File has exactly one rule (no extra lines that could conflict)
</acceptance_criteria>
</task>

```bash
cd /Users/petr/projects/kindlm

npx wrangler pages deploy packages/dashboard/out --project-name=kindlm-dashboard
```

If the Pages project does not exist yet, create it first:
```bash
npx wrangler pages project create kindlm-dashboard
```

After deployment, Wrangler will output the deployment URL (e.g., `https://abc123.kindlm-dashboard.pages.dev`).

**Custom domain setup** (if `app.kindlm.com` is not yet connected):

The custom domain can be configured in the Cloudflare dashboard under Pages > kindlm-dashboard > Custom domains > Add custom domain > `app.kindlm.com`.

Alternatively, add a CNAME DNS record:
```bash
TOKEN="$(grep CLOUDFLARE_API_TOKEN ~/projects/ai-news-monitor/.env.local | cut -d= -f2)"
ZONE=$(curl -s "https://api.cloudflare.com/client/v4/zones?name=kindlm.com" \
  -H "Authorization: Bearer $TOKEN" | python3 -c "import json,sys; print(json.load(sys.stdin)['result'][0]['id'])")

# Create CNAME for app.kindlm.com -> kindlm-dashboard.pages.dev
curl -s -X POST "https://api.cloudflare.com/client/v4/zones/$ZONE/dns_records" \
  -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  -d '{"type":"CNAME","name":"app.kindlm.com","content":"kindlm-dashboard.pages.dev","proxied":true}'
```

Then add `app.kindlm.com` as a custom domain in Cloudflare Pages dashboard.
</action>

# DASH-03: Hashed asset has immutable header (find a real asset URL first)
ASSET_PATH=$(curl -s "$DASH_URL/" | grep -o '/_next/static/[^"]*\.js' | head -1)

fi

- **Custom domain 522/523:** DNS propagation may take up to 5 minutes. Wait and retry.
