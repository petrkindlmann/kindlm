    **src/junit.ts:** Create with two exports:

    1. `generateJunitXml(report: KindlmJsonReport): string`
       - Build JUnit XML string manually (no XML library needed for this simple structure):
         ```xml
         <?xml version="1.0" encoding="UTF-8"?>
         <testsuites name="KindLM" tests="{total}" failures="{failed}" errors="{errored}" time="{durationMs/1000}">
           {For each suite:}
           <testsuite name="{suite.name}" tests="{suite.tests.length}" failures="{failCount}">
             {For each test:}
             <testcase name="{test.name}" classname="{suite.name}">
               {If test.status !== 'passed':}
               <failure message="{first failed assertion label}">{assertion labels joined by newline}</failure>
             </testcase>
           </testsuite>
         </testsuites>
         ```
       - CRITICAL (D-19): Only include `test.name`, `suite.name`, `assertion.label`, and `assertion.failureMessage` in XML. NEVER include model response text. XML-escape all string values (`&`, `<`, `>`, `"`, `'`).

       - Wrap in try/catch; on failure: `core.warning(`Artifact upload failed: ${err}`)` — non-fatal
