**Multi-file watching (WATCH-01, per D-03):**
- Read the config file content (already done for isolate), parse with `extractConfigFilePaths(yamlContent)`.
- Resolve referenced paths against `dirname(configPath)` (NOT cwd — per research pitfall 3).
- Build `watchPaths = [configPath, ...resolvedReferencedPaths]`.
- Pass `watchPaths` to `watchFiles()`.

**Abort logic (WATCH-02, per D-05, D-06, D-07):**
- Declare module-level state in the watch block: `let runInProgress = false`, `let pendingRerun = false`, `let abortRef = { aborted: false }`.
- `triggerRun` async function:
  - If `runInProgress`: set `abortRef.aborted = true`, set `pendingRerun = true`, return.
  - Set `runInProgress = true`, create fresh `abortRef = { aborted: false }`.
  - Print separator if `sessionRuns > 0` (per D-08): `console.log(chalk.dim(\`\u2500\u2500\u2500\u2500 [\${new Date().toISOString()}] \u2500\u2500\u2500\u2500\`))`.
  - Call `executeTestRun(options)` in try/catch.
  - After run: increment `sessionRuns`, extract and accumulate cost.
  - Print cost line: `console.log(chalk.dim(\`Session cost: $\${sessionCostUsd.toFixed(4)} (\${sessionRuns} run\${sessionRuns === 1 ? "" : "s"})\`))` (per D-12).
  - Set `runInProgress = false`.
  - If `pendingRerun`: set `pendingRerun = false`, call `void triggerRun()`.
- NOTE: `executeTestRun` does NOT currently accept an abort signal. For this phase, the abort flag prevents queuing multiple runs. The in-progress run completes its current test naturally. This is sufficient for D-05/D-06 — the run finishes its current test but a new run won't stack on top.

**Cost extraction (WATCH-05, per D-11, D-12, D-13):**
- `RunResult` has NO `totalCostUsd`. Extract cost from: `result.suites.flatMap(s => s.tests).reduce((sum, t) => sum + (t.costUsd ?? 0), 0)`.
- To access the RunResult from inside the watch loop, modify `executeTestRun` to return the cost. Add a return type: the function currently returns `Promise<void>`. Change it to return `Promise<{ costUsd: number; passed: number; failed: number } | undefined>`.
- Inside `executeTestRun`, before the exit-code block, compute cost: `const totalCostUsd = result.suites.flatMap(s => s.tests).reduce((sum, t) => sum + (t.costUsd ?? 0), 0)`.
- In watch mode (already skips `process.exit`), return `{ costUsd: totalCostUsd, passed: result.passed, failed: result.failed }`.
- In non-watch mode, behavior unchanged (process.exit before return).
- In catch blocks, return `undefined`.

**Separator output (WATCH-04, per D-08, D-09):**
- Print dim separator with ISO timestamp before each re-run (not the first run).
- Do NOT clear terminal (per D-09).

**Signal handling (WATCH-06, per D-14, D-15, D-16):**
- After setting up the file watcher, register cleanup:
  ```
  let cleanedUp = false;
  const cleanup = () => {
    if (cleanedUp) return;
    cleanedUp = true;
    abortRef.aborted = true;
    fileWatcher.close();
    console.log(chalk.dim(`\nWatch session ended. ${sessionRuns} run${sessionRuns === 1 ? "" : "s"}, $${sessionCostUsd.toFixed(4)} total cost.`));
    process.exit(0);
  };
  process.on("SIGINT", cleanup);
  process.on("SIGTERM", cleanup);
  ```
- Guard with `cleanedUp` boolean (per D-16) to prevent double-execution.

**Cache integration (WATCH-07, per D-17):**
- No special work needed. The existing `createCachingAdapter` in `run-tests.ts` handles caching. Config-only changes (tweaking assertions) will hit cache automatically.

**Existing "Watching..." message (per D-10):**
- Keep the existing console.log for "Watching ... for changes..." but update to mention the number of watched files if more than 1.

**IMPORTANT anti-patterns to avoid:**
- Do NOT use chokidar 5.x.
- Do NOT clear terminal between runs.
- Do NOT stack concurrent runs (max 1 pending re-run queued).
- Do NOT call process.exit() before fileWatcher.close() in cleanup.
