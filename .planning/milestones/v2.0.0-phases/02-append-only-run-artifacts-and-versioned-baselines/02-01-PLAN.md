    **2c — Immutable baseline versioning in `store.ts`:**

    Add `savedAt` field to `BaselineData` (optional to preserve backward compat with v1 files already on disk — existing `deserializeBaseline` does not require it):

    ```typescript
    export interface BaselineData {
      version: string;
      suiteName: string;
      createdAt: string;
      savedAt?: string;       // ISO timestamp added on write (immutability marker)
      results: Record<string, BaselineTestEntry>;
    }
    ```

    Add a new `BaselineVersionedIO` interface extending `BaselineIO` with a `writeVersioned` method. Do NOT change `BaselineIO` — it is implemented by `cli/src/utils/baseline-io.ts` and must stay stable.

    Instead, add a new exported function `writeBaselineVersioned(data: BaselineData, io: BaselineIO): Result<void>` that:
    1. Stamps `data.savedAt = new Date().toISOString()` (non-mutating — create a new object: `const stamped = { ...data, savedAt: new Date().toISOString() }`).
    2. Computes `timestamp = stamped.savedAt.replace(/[-:T.Z]/g, "").slice(0, 14)` (produces `YYYYMMDDHHMMSS`).
    3. Calls `io.write(`${data.suiteName}-${timestamp}`, JSON.stringify(stamped))` to write the versioned copy.
    4. Calls `io.write(`${data.suiteName}-latest`, JSON.stringify({ latestFile: `${data.suiteName}-${timestamp}.json` }))` — pointer file contains ONLY the filename reference, NOT a content copy. This is the locked architectural decision.
    5. Returns the result of the latest-pointer write. If the timestamped write fails, return that error immediately.

    The existing `writeBaseline` function stays unchanged (used internally). The `kindlm baseline set` command (`packages/cli/src/commands/baseline.ts`) should be updated to call `writeBaselineVersioned` instead of `writeBaseline`.

    Check `packages/cli/src/commands/baseline.ts` and `packages/cli/src/utils/baseline-io.ts` to verify the call site — update the `set` subcommand to use `writeBaselineVersioned`. Also update `baseline-io.ts` if the `read` method needs to fall back to `-latest` suffix when the plain `suiteName` file doesn't exist.
